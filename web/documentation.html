Documentation

An overview for GameR template

GameR is an ideal template for game-related websites (gaming, communities, clans, guilds, news and entertainment content) but also for technology, music, movies, CV or personal pages. Smart and compact. It's able to completely meet your needs. Many advanced customizable sections will help you get your own website fast and easy. Unique design will increase the value of your page!

User gets many unique components like timeline, counter, charts, requirements boxes, calendar to use. You can easily personalize them to your specific goals.

What's included

GameR template includes pages:

  • index.html (main one page scroll)
  • index-no-animation.html (main page without skrollr animations)
  • blog-list.html (single page with posts list)
  • blog-item.html (single page with post item)
  • typography.html (single page with bootstrap components)

...and also HTML files for every color variant.

Pages validate correctly in the W3C validator and are fully compatible with HTML5.

LESS

General

  1. Custom LESS code is included in less/custom subfolder.
  2. Font Awesome LESS code is included in assets/font-awesome-4.7.0/less/font-awesome.less. This files is imported in less/bootstrap.less file.
  3. Morris chart LESS code is included in assets/morris.js-0.5.1/less/morris.core.less. This files is imported in less/bootstrap.less file.
  4. less/custom subfolder contains three files: mixins.less, styles.less and variables.less. These three files are imported in less/bootstrap.less file.
  5. Every modification of LESS code should be appended to less/custom/styles.less or less/custom/variables.less file.
  6. If you want to create your own LESS mixin you can add it to the less/custom/mixins.less file.
  7. After any modification you should compile less/bootstrap.less to css/styles.css and css/styles.min.css files - you may use the grunt for this.
  8. If you want to change color variant for template you have to edit the less/custom/variables.less file. There are some variables that you have to change:
    • @main-color
    • @background-edge-url
    • @background-edge-small-url
    • @background-pixel-url
  9. You may choose between 6 predefined colors:
    • @yellow
    • @orange
    • @pink
    • @red
    • @green
    • @blue
  10. For example if you want to change the template color for red you have to type: 
    @main-color:                    @red;
    @background-edge-url:           "../img/home-bg-red.png";
    @background-edge-small-url:     "../img/single-page-header-bg-red.png";
    @background-pixel-url:          "../img/bg-red.jpg";
  11. As you can see above, last 3 parameters are paths to specific image files. You have to change the filenames replacing old color for the new one: 
    "../img/home-bg-yellow.png"   --->   "../img/home-bg-red.png"
  12. Next you have to compile less/bootstrap.less to css/styles.css and css/styles.min.css files - you may use the grunt for this.
  13. At the end you have to attach your CSS file in document head section, example: 
    <link href="css/styles.min.css" media="screen" rel="stylesheet" type="text/css">

Editing styles

You can create additional LESS file or edit less/custom/styles.less and less/custom/variables.less files and after that compile less/bootstrap.less into the css/styles.css and css/styles.min.css files - you may use grunt for this. If you want to create new LESS file don't forget to import it at the end of less/bootstrap.less file. Example:

// GameR styles
@import "custom/styles.less";
//your own styles
@import "custom/your-styles.less";

    • css/red.css, css/red.min.css
    • css/green.css, css/green.min.css
    • css/blue.css, css/blue.min.css
  3. You can attached both css/styles.css (css/styles.min.css) or one of the color variation above. By default all the LESS code is compiled to the css/styles.css and css/styles.min.css files. In other words you can change the styles and color variation in two ways:
    • You can manipulate LESS code.
    • You can choose CSS file corresponding to the specific color.
  4. Lightbox2 CSS code is defined in assets/lightbox2-master/dist/css/lightbox.css and assets/lightbox2-master/dist/css/lightbox.min.css files.
  5. loaders.css-master CSS code is defined in assets/loaders.css-master/loaders.css and assets/loaders.css-master/loaders.min.css files.
  6. All CSS files are available in both versions - uncompressed and minified.
  7. css/styles.css, css/styles.min.css and all colors variants files are compiled from LESS code.

Variant selection

  1. In the package of GameR template you may find separate HTML files for every color variant, but sometimes you may need to edit some other file with all colors variants included.
  2. In this case you should remove some HTML <link> tags and replace them with one HTML <link> tag which attaches the specify color variant.
  3. In most of the pages you can find the code: 
    <link href="css/yellow.min.css" media="screen" rel="stylesheet" type="text/css" title="yellow">
<link href="css/orange.min.css" media="screen" rel="alternate stylesheet" type="text/css" title="orange">
<link href="css/pink.min.css" media="screen" rel="alternate stylesheet" type="text/css" title="pink">
<link href="css/red.min.css" media="screen" rel="alternate stylesheet" type="text/css" title="red">
<link href="css/green.min.css" media="screen" rel="alternate stylesheet" type="text/css" title="green">
<link href="css/blue.min.css" media="screen" rel="alternate stylesheet" type="text/css" title="blue">
  4. You should choose the color variant you want and remove the other HTML <link> tags. For example, if you want to have a red variant on your page, remove all the code above instead of this line: 
    <link href="css/red.min.css" media="screen" rel="alternate stylesheet" type="text/css" title="red">
  5. Next you have to change the rel attribute value of HTML <link> tag to "stylesheet" (it can't be "alternate stylesheet"). Example: 
    <link href="css/red.min.css" media="screen" rel="stylesheet" type="text/css" title="red">
  6. There are two possible values of rel attribute:
    • rel="stylesheet" - for main color
    • rel="alternate stylesheet" - for every alternative color (for color switcher)
  7. Finally you should not have any CSS link element with rel="alternate stylesheet" (excepts you want to have a color switcher in you page).
  8. If you preserve the morris charts in your page, you have to also change their initial color values. To do that you should change the data attributes for HTML chart's elements: 
    <div
    class="chart chart-circle no-margin"
    id="order-chart"
    data-chart-color-background="#292929"   //background color of current chart
    data-chart-color-first="#FFD500"        //first color of current chart
    data-chart-color-second="#D9AF00"       //second color of current chart
    data-chart-color-third="#B38900"        //third color of current chart
></div>
    There are the initial values used for canvas drawing.
  9. At least remove the HTML code of color switcher: 
    <div class="sidebar">
    <a class="sidebar-switcher" href="#" title="Color switcher">
        <i class="fa fa-gear"></i>
    </a>
    <div class="sidebar-content">
        <h4 class="sidebar-header">Color switcher</h4>
        <a href="#" class="color-switcher bg-yellow" data-color="yellow"></a>
        <a href="#" class="color-switcher bg-orange" data-color="orange"></a>
        <a href="#" class="color-switcher bg-pink" data-color="pink"></a>
        <a href="#" class="color-switcher bg-red" data-color="red"></a>
        <a href="#" class="color-switcher bg-green" data-color="green"></a>
        <a href="#" class="color-switcher bg-blue" data-color="blue"></a>
    </div>
</div>

Editing styles

If you want to edit CSS styles you can create additional CSS file with your own CSS code and include this file in page that needs it after any others <link> tags: 

<link href="css/yellow.css" media="screen" rel="stylesheet" type="text/css" title="yellow">
<link href="css/your_own_styles.css" media="screen" rel="stylesheet" type="text/css">

Loaders.css.master

GameR use the delightful and performance-focused pure css loading animations. This view is showing until all the assets will be loaded. loaders.css-master CSS code are defined in assets/loaders.css-master/loaders.css and assets/loaders.css-master/loaders.min.css files.

If you want to remove CSS preloader logic from your page you have to remove some code:

  1. In js/custom.js (or js/custom.min.js) file: 
    //preloader code start
var preloader = $('.loader-overlay');
preloader.fadeOut(1000, function () {
    $(this).remove();
});
//preloader code end
  2. In index.html (or index-no-animations.html) file: 
    <div class="loader-overlay">
    <div class="loader">
        <div class="loader-inner pacman">
            <div></div>
            <div></div>
            <div></div>
            <div></div>
            <div></div>
        </div>
        <h3 class="loader-text">Loading>/h3>
    </div>
</div>
  3. You can also remove the <link> tag attaching the CSS file for this loader (index.html or index-no-animations.html): 
    <!-- Loaders styles -->
<link href="assets/loaders.css-master/loaders.min.css" media="screen" rel="stylesheet" type="text/css">

More information here: https://github.com/ConnorAtherton/loaders.css

General

  1. All the custom JavaScript for GameR template is written in strict mode.
  2. All the custom JavaScript code is included in js/custom.js and js/custom.min.js files. You can add your own code there.
  3. All the js code sections have start and end comment. Example: 
    //background parallax code start
if (!Modernizr.touchevents) {
    $('#home').parallax("0%", 0.2, true);
    $('#buy').parallax("0%", 0.2, true);
    $('#flow').parallax("0%", 0.2, true);
    $('#testimonials').parallax("0%", 0.2, true);
    $('#counter').parallax("0%", 0.2, true);
    $('#partners').parallax("0%", 0.2, true);
    $('#send-us-message').parallax("0%", 0.2, true);
}
//background parallax code end
  4. If you want to remove background parallax logic you have to remove all the code between //background parallax code start and //background parallax code end comments.
  5. If you are sure that parallax plugin is not used in any other place of your page you can also remove the <link> tag (index.html or index-no-animation.html page), which attached this plugin on page: 
    <!-- jQuery parallax -->
<script type="text/javascript" src="js/jquery.parallax-1.1.3-min.js"></script> //this code should be removed if you don't use parallax in any place
  6. Sometimes you have to also remove some of the HTML code to disable specific functionality. Example:

    In js/custom.js (or js/custom.min.js) file: 
    //preloader code start
var preloader = $('.loader-overlay');
preloader.fadeOut(1000, function () {
    $(this).remove();
});
//preloader code end
    In index.html (or index-no-animations.html) file: 
    <div class="loader-overlay">
    <div class="loader">
        <div class="loader-inner pacman">
            <div></div>
            <div></div>
            <div></div>
            <div></div>
            <div></div>
        </div>
        <h3 class="loader-text">Loading>/h3>
    </div>
</div>
  7. If you want to remove preloader (animation when document is loading), you have to remove all the code between //background parallax code start and //background parallax code end comments. You have to also remove the div.loader-overlay element (this one above in index.html file) with all his children elements. You can also remove the attaching CSS file for this loader: 
    <!-- Loaders styles -->
<link href="assets/loaders.css-master/loaders.min.css" media="screen" rel="stylesheet" type="text/css">

Lightbox2

This plugin allows user to show images on new layer (it is used in gallery section).

If you want to disable the lightbox follow the instructions:

  1. Remove every data-lightbox (also data-title) attribute from every place in HTML code (especially from gallery items). Example: 
    <a class="portfolio-caption-icon" data-lightbox="portfolio" data-title="Minecraft" href="img/screenshot-1.jpg" title="Zoom"><i class="fa fa-search-plus" aria-hidden="true"></i></a>
    has to be changed to: 
    <a class="portfolio-caption-icon" href="img/screenshot-1.jpg" title="Zoom"><i class="fa fa-search-plus" aria-hidden="true"></i></a>

    <!-- Lightbox styles -->
<link href="assets/lightbox2-master/dist/css/lightbox.min.css" rel="stylesheet">
  3. Remove the <script> tag from index.html or index-no-animation.html if you are sure that you don't use lightbox2 in any other place: 
    <!-- Lightbox -->
<script type="text/javascript" src="assets/lightbox2-master/dist/js/lightbox.min.js"></script>

jQuery Parallax

Plugin used for background animations. Used in home, buy/promo, getting started, testimonials, coming soon, partners and send us message sections. This feature is disabled for touch devices (for better performance and compability). More info here: https://github.com/IanLunn/jQuery-Parallax

skrollr

This plugin allows user to animate any CSS property of any element depending on the horizontal scrollbar position. User can use the HTML5 data- attributes to define multiple sets of styles (call each of them keyframe) and skrollr interpolates between them. GameR template contains two version of main page: with and without skrollr animations. If the performance is the priority for you, you should consider using variant without animations. This feature is always disabled for touch devices regardless of the page variant (for better performance and compability).

If you want to remove skrollr logic you can do it in that way:

  1. Remove class="skrollr" and data-skrollr="true" from <body> element: 
    <body class="skrollr" data-skrollr="true">...</body>  --->  <body>...</body>
  2. It is enough for disabling skrollr logic but there is a good practice to make some clean up in a HTML code.
  3. Remove all the data elements connected with skrollr plugin: 
    <span class="plus" data-100p-top="transform[swing]:scale(0.2);opacity[swing]:0" data-75p-top="transform[swing]:scale(1);opacity[swing]:1">+</span>   --->   <span class="plus">+</span>
  4. If you are sure that you don't use skrollr somewhere else, you can also remove the <link> tag attaching the JavaScript code for this plugin (index.html or index-no-animation.html). 
    <!-- skrollr -->
<script type="text/javascript" src="js/skrollr.min.js"></script>
  5. More specific information available at: https://github.com/Prinzhorn/skrollr

countdown.js

countdown.js is an ultra-light jQuery countdown timer plugin which counts down from a specific date and injects countdown elements (days, hours, minutes, seconds) into selected elements for easier styling. In this template this library is used in coming soon section.

You can easily customize this plugin for your needs:

//counter code start
var someDate = new Date();
var dd = ("0" + (someDate.getDate() + 3) ).slice(-2);
var mm = ("0" + (someDate.getMonth() + 1) ).slice(-2);
var y = someDate.getFullYear();
var countdownDate = y + '/' + mm + '/' + dd;
$('#clock').countdown({
    date: countdownDate
});
//counter code end

Replace the code above by this:

//counter code start
var countdownDate = '5 september 2020 23:5'; // here goes your date
$('#clock').countdown({
    date: countdownDate
});
//counter code end

For details please see the official documentation: http://www.jqueryscript.net/time-clock/Minimalist-jQuery-Countdown-Timer-Plugin-countdown-js.html
